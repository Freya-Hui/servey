# 请上传这个文件夹里的内容

你需要上传的是本文件夹里面的内容，而不是上传外层文件夹本身。

上传到 GitHub 后，GitHub 仓库首页应该直接看到：

```text
index.html
media
请先看这里_上传说明.md
```

## 必须上传

- `index.html`
- `media` 整个文件夹

## 不要只上传压缩包

如果你只上传 `.zip`，Vercel 不会自动解压，会找不到网页。

## Vercel 设置

- Framework Preset：Other
- Build Command：留空
- Output Directory：留空
- Install Command：留空
- Root Directory：留空

## 如果仍然 404

检查 GitHub 仓库是否是这样的结构：

```text
你的仓库/
  index.html
  media/
```

如果变成下面这样，就会出错：

```text
你的仓库/
  【上传到GitHub的最终版本】/
    index.html
    media/
```

如果是这种情况，要么把 `index.html` 和 `media` 移到仓库最外层，要么在 Vercel 的 Root Directory 里填写：

```text
【上传到GitHub的最终版本】
```
